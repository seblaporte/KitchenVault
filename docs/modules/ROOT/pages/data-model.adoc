== Tables de jointure

* `recipe_category` : liaison Recipe ↔ Category (many-to-many)
* `chapter_recipe` : liaison Chapter ↔ Recipe avec `sort_order` (many-to-many ordonné)
* `recipe_note` : collection de notes texte par recette
* `recipe_utensil` : collection d'ustensiles par recette

== Stratégie d'upsert

Les identifiants Cookidoo (`id`) sont utilisés comme clés primaires pour permettre un upsert natif via `save()` JPA :

* Si la recette est absente → INSERT
* Si la recette est présente et son `last_synced_at` est plus ancien que le seuil configuré (`cookidoo.sync.resync-after-hours`) → UPDATE complet
* Si la recette est fraîche → pas de requête vers Cookidoo
